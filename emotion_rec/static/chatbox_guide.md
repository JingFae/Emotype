# Emo 回响 · Chatbox 使用说明

**Emo 回响**（Emo Echo）是 EmoBridge 的 AI 情感陪伴聊天模块，提供温柔、非诊断式的情绪倾听与互动。

---

## 功能概览

| 功能 | 说明 |
|------|------|
| 实时 AI 对话 | 由 DeepSeek 大语言模型驱动，温柔回应你的情绪诉说 |
| 历史会话存储 | 每次对话自动保存，可随时回顾或继续 |
| 个人情绪上下文 | 登录后，AI 会参考你近 15 天的日记、随手记、身体感受 |
| 多语言支持 | 界面文字随系统语言设置（中文 / English）自动切换 |
| 新对话 | 一键开启全新会话，不影响历史记录 |

### 按钮说明

- **历史记录**：打开侧边面板，查看所有历史会话列表，点击任意一条可恢复对话内容
- **新对话**：清空当前消息区域，生成新的会话 ID，开始全新对话
- **发送按钮 ➤**：发送消息（也可按 Enter 键）
- **关闭**：关闭历史对话面板

## 使用方法

### 基础对话

1. 在底部输入框输入你想说的内容
2. 按 **Enter** 或点击 **➤** 发送
3. AI 会在 3-5 句内温柔回应，并适当提出一个引导问题
4. 使用 **Shift + Enter** 可在输入框内换行

### 查看历史记录

1. 点击右上角 **历史记录** 按钮
2. 侧面板显示所有会话，包含时间、第一条消息预览和消息数量
3. 点击某一条会话，页面自动恢复该对话的完整内容
4. 点击 **关闭** 收起面板

### 开始新对话

点击 **新对话** 按钮，当前页面的消息记录会被清空，生成新会话 ID。历史数据不受影响，可通过「历史记录」面板随时查看。

## AI 回复规则

Emo 回响遵循以下设计原则：

- **温暖口语化**：不过度正式，像一位有见识的朋友
- **不轻易下结论**：多使用「好像」「看起来」「也许」等措辞
- **不做诊断**：不评判、不建议停药，不替代专业心理帮助
- **适当引导**：每次回复会提出一个温柔的开放性问题
- **控制长度**：每次回复约 3-5 句，不冗长

## 个人上下文（需登录）

登录账户后，Emo 回响会在每次对话时自动读取你**近 15 天**的情绪数据作为背景参考：

| 数据来源 | 内容 |
|---------|------|
| 正式日记 | 日期、情绪标签、内容摘要（前 100 字） |
| 情绪随手记 | 日期、情绪标签、文字内容（前 80 字） |
| 身体感受 | 选择的身体部位和感受描述 |

> AI 会轻轻呼应这些记录，但不会机械引用原文，也不会将其暴露在对话中。

未登录时，对话以游客身份进行，数据存储在本地 `participant_code = local`，不与账户绑定。

## 技术架构

```
前端 (emo_echo.html / emo_echo.js)
    │
    │ POST /api/emo-echo/chat
    ▼
后端 (app.py · emo_echo_chat)
    │── get_echo_context()  ← 读取日记/随手记/身体感受
    │── llm_client.chat()   ← 调用 DeepSeek API
    │── save_echo_message() ← 持久化到 SQLite
    ▼
返回 { reply: "...", session_id: "..." }

GET /api/emo-echo/sessions → 历史会话列表
```

### 关键配置

| 环境变量 | 默认值 | 说明 |
|---------|-------|------|
| `DEEPSEEK_API_KEY` | 见代码 | DeepSeek API 密钥 |
| `DEEPSEEK_MODEL` | `deepseek-v4-flash` | 使用的模型 |
| `DEEPSEEK_BASE_URL` | `https://api.deepseek.com` | API 端点 |
| `GEMINI_API_KEY` | 见代码 | Google AI Studio 密钥，用于图片上传的 Gemini VLM 分析 |
| `GEMINI_MODEL` | `gemini-2.5-flash` | 图片分析使用的 Gemini 模型 |
| `LLM_ENABLED` | `true` | 设为 `false` 可完全禁用 LLM |

LLM 不可用时，系统自动使用本地兜底回复，不影响页面正常显示。

## 数据存储

| 表 | 内容 |
|----|------|
| `echo_sessions` | 会话记录（session_uuid、participant_id、创建/最后活跃时间） |
| `echo_messages` | 每条消息（session_id、role、content、时间戳） |

所有数据存储在本地 SQLite 文件 `emomirror_data.sqlite3`，不上传到第三方服务器。

---

## 注意事项

- Emo 回响是**情感陪伴工具**，不是心理治疗或诊断系统
- 如有严重心理困扰，请寻求专业帮助
- 对话内容仅存储在你的服务器本地，不用于模型训练
- 每次会话历史最多保留最近 **20 条**消息作为上下文，更早的内容不参与本次对话
