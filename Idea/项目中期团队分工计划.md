# EmoType / EmoMirror 项目中期团队分工计划

本文档用于组内中期阶段的任务分配。分工依据当前项目 README、现有代码结构和后续优化方向制定，目标是让 4 位人工智能技术同学和 2 位设计专业同学都能形成清晰、可展示、可合并到项目中的贡献。

如果后续组员姓名确定，可以直接将文中的 `AI-1`、`AI-2`、`AI-3`、`AI-4`、`Design-1`、`Design-2` 替换为真实姓名。

## 一、总体协作目标

中期后项目应继续围绕以下主线推进：

- 提升音频和文本情绪识别的可靠性。
- 优化 V-A 情绪映射、标签、颜色和动态字体规则。
- 改善前端交互体验，使系统更像一个完整的情绪日记应用。
- 完善后端数据库、数据导出、部署和测试流程。
- 让设计同学的 Figma/Canva 产出能被真正嵌入网页 UI，而不是停留在展示图。

## 三、人工智能技术同学详细任务

### AI-1：音频情绪识别与模型评估

负责让“语音输入到 VAD 输出”这条路径更加可信。

当前基础：

- `/predict` 已经可以接收音频文件。
- 本地 `Wav2vec-2.0/` 模型会输出 `arousal`、`dominance`、`valence`。
- 后端会提取 pitch、energy 等声学特征。

近期任务：

- 整理 `audio/` 中已有样本，形成一份音频测试清单。
- 分别测试 WAV、WebM、MP3 或浏览器录音格式的输入表现。
- 记录不同音频的模型输出，观察 VAD 是否符合直觉。
- 检查 `VAD_SOURCE_RANGE` 是否适合当前模型输出。
- 输出一份《音频情绪识别测试记录》，包含样本、预期情绪、模型输出和问题分析。

后续可扩展：

- 尝试模型量化或推理加速。
- 增加模型加载失败时的友好提示。
- 研究是否需要重新训练或微调音频情绪模型。

### AI-2：文本情绪识别与 V-A 映射

负责让“文本内容到情绪标签/V-A 坐标”这条路径更加合理。

当前基础：

- `/analyze-text` 已经调用 `text_emotion.py`。
- 文本分析支持规则 fallback、中文情绪分类器和未来 regression head。
- `va_mapper.py` 负责把 V-A 坐标映射到标签、颜色、象限和候选情绪。
- `emotion_lexicon.json` 中已有 80 个情绪标签。

近期任务：

- 阅读并整理 `text_emotion.py` 中已有规则，说明每类规则识别什么情绪线索。
- 设计 20-40 条中文测试句，覆盖开心、焦虑、愤怒、悲伤、平静、矛盾表达等情况。
- 测试每条文本的 `text_emotion.segments` 和 `va_mapping.overall` 是否合理。
- 优化 `emotion_lexicon.json` 中明显不合理的标签位置或命名。
- 与 Design-2 一起确定四象限颜色和情绪标签表达。

后续可扩展：

- 准备文本 V-A 标注数据。
- 训练并接入 `emotion_rec/models/text_emotion_head.pt`。
- 让用户手动修改后的情绪标签反向用于规则校准。

### AI-3：前端交互实现与设计落地

负责把设计同学的界面和素材真正接入当前 Web UI。

当前基础：

- 前端在 `emotion_rec/static/`。
- `index.html` 提供页面结构。
- `styles.css` 提供视觉样式和动画。
- `app.js` 负责文本分析请求、日记保存、V-A 坐标拖拽、动态字体渲染。

近期任务：

- 和 Design-1 对齐新版首页和 Journal 页面的 Figma 设计稿。
- 将设计稿拆解为可实现的 UI 区块：顶部导航、日记输入区、情绪镜像区、V-A 面板、候选标签、保存导出区。
- 将 Design-2 提供的素材放入 `emotion_rec/static/assets/`。
- 在 HTML/CSS 中引用 SVG、PNG、WebP 等素材。
- 修复当前页面中的中文乱码文案。
- 完善前端状态：分析中、分析失败、保存成功、无输入、无网络。

后续可扩展：

- 增加移动端布局。
- 增加调试面板，显示当前 segment、V-A、evidence 和 design map。
- 如果前端继续变复杂，可评估是否迁移到组件化框架。

### AI-4：后端数据库、部署与工程化

负责让项目能稳定保存数据、导出数据、部署运行和被组员协作维护。

当前基础：

- `storage.py` 已经定义参与者、日记、使用事件三类数据。
- 默认 SQLite，可通过 `DATABASE_URL` 切换 PostgreSQL。
- `Dockerfile` 和 `render.yaml` 已支持 Render 部署。
- `DEPLOYMENT.md` 已记录部署说明。

近期任务：

- 梳理所有 API 接口，形成一份接口表。
- 测试参与者登录、日记保存、个人导出、管理员导出是否正常。
- 检查本地 SQLite 与 Render PostgreSQL 的环境变量配置差异。
- 增加基础测试流程，例如运行 `python -m compileall emotion_rec emotion_computing hmotiongpt-api-test`。
- 明确哪些文件不能提交：`.env`、证书、私钥、日志、缓存、本地数据库。

后续可扩展：

- 引入 Alembic 做数据库迁移。
- 配置 GitHub Actions 自动检查。
- 优化 Render 部署内存和模型文件管理。
- 补充研究数据隐私说明和删除机制。

## 四、设计专业同学详细任务

### Design-1：UI/UX 与 Figma 页面设计

负责界面结构、用户流程和交互状态。

近期任务：

- 在 Figma 中重设计两个核心页面：
  - Home 页面：项目入口、最近日记、情绪概览。
  - Journal 页面：文本输入、情绪镜像、候选标签、V-A 坐标、保存导出。
- 为每个页面提供桌面端和移动端版本。
- 设计以下状态：
  - 默认状态
  - 输入中
  - 分析中
  - 分析成功
  - 分析失败
  - 保存成功
  - 本地模式和连接参与者编号后的数据库模式
- 建立基础组件库：
  - button
  - input
  - textarea
  - tag/chip
  - slider
  - tab
  - emotion card
  - V-A coordinate panel

交付格式：

- Figma 页面链接。
- 页面截图 PNG。
- 组件命名和使用说明。
- 每个交互状态的说明。

### Design-2：情绪视觉系统与动态字体规范

负责“情绪如何被看见”。

近期任务：

- 为 V-A 四象限设计视觉规范：
  - 积极高能量
  - 积极低能量
  - 消极高能量
  - 消极低能量
  - 中性
- 输出情绪视觉映射表：

| 情绪象限 | 色彩 | 字体感觉 | 动画感觉 | 视觉关键词 |
| --- | --- | --- | --- | --- |
| 积极高能量 | 橙、黄、粉 | 粗、放大、跳跃 | pulse-scale | 兴奋、惊喜、明亮 |
| 积极低能量 | 绿、青、浅蓝 | 稳定、柔和 | float-drift | 安心、放松、舒适 |
| 消极高能量 | 红、深红、暗紫 | 紧张、厚重、压迫 | shake-hard | 愤怒、焦虑、紧绷 |
| 消极低能量 | 蓝、灰、深青 | 轻、下沉、留白 | sad-droop | 悲伤、疲惫、孤独 |
| 中性 | 灰、浅蓝灰 | 清晰、平衡 | 轻微浮动或无动画 | 平稳、观察、未明确 |

- 设计可嵌入项目的素材：
  - 情绪 icon
  - emoji 替代图形
  - 背景纹理
  - 情绪镜像装饰元素
  - V-A 平面视觉元素

交付格式：

- SVG：logo、icon、表情、坐标图形。
- WebP/PNG：背景图、纹理、插画。
- Lottie JSON 或 GIF：复杂动效，若后续要加入。
- 一份素材说明表，说明每个素材的用途和触发场景。

## 五、设计素材如何嵌入当前 UI

设计素材建议统一放在：

```text
emotion_rec/static/assets/
```

推荐目录：

```text
emotion_rec/static/assets/
├── icons/
├── emotions/
├── backgrounds/
├── illustrations/
├── textures/
└── motion/
```

推荐命名方式：

```text
emotion-positive-high.svg
emotion-positive-low.svg
emotion-negative-high.svg
emotion-negative-low.svg
emotion-neutral.svg
mirror-texture-soft.webp
va-plane-grid.svg
empty-journal-illustration.webp
```

HTML 中引用：

```html
<img src="/static/assets/emotions/emotion-positive-high.svg" alt="积极高能量情绪" />
```

CSS 中引用：

```css
.mirror-surface {
  background-image: url("/static/assets/backgrounds/mirror-texture-soft.webp");
}
```

JavaScript 中根据情绪象限切换素材：

```js
const emotionAssetMap = {
  high_positive: "/static/assets/emotions/emotion-positive-high.svg",
  low_positive: "/static/assets/emotions/emotion-positive-low.svg",
  high_negative: "/static/assets/emotions/emotion-negative-high.svg",
  low_negative: "/static/assets/emotions/emotion-negative-low.svg",
  neutral: "/static/assets/emotions/emotion-neutral.svg",
};
```

## 六、设计交付给技术同学时需要包含什么

每次设计交付建议包含：

1. Figma 页面链接。
2. 页面截图。
3. 组件状态说明。
4. 色彩、字体、间距参数。
5. 导出的素材文件。
6. 素材用途说明。
7. 哪些内容是必须实现，哪些是可选优化。

设计同学给出的颜色和间距，建议最后整理成 CSS 变量：

```css
:root {
  --emotion-high-positive: #F59E0B;
  --emotion-low-positive: #10B981;
  --emotion-high-negative: #DC2626;
  --emotion-low-negative: #2563EB;
  --emotion-neutral: #94A3B8;
}
```

这样技术同学可以把设计规范同步到 `styles.css`、`va_mapper.py` 和 `vaMapper.js` 中。

## 七、每周协作流程建议

每周建议固定一次 30-45 分钟同步：

1. AI-1 汇报音频模型测试结果。
2. AI-2 汇报文本情绪和 V-A 映射测试结果。
3. Design-1 展示 UI 页面或交互状态更新。
4. Design-2 展示情绪视觉系统或素材更新。
5. AI-3 说明哪些设计已经落地到前端，哪些还缺素材或规则。
6. AI-4 说明当前保存、导出、部署和测试是否稳定。
7. 最后共同确认下一周要合并到项目的内容。

## 八、中期到后期的里程碑

### 第 1 阶段：整理和对齐

- 完成 README 和本分工文档。
- 每位组员明确自己的模块。
- 设计同学完成第一版 Figma 页面和情绪视觉表。
- 技术同学完成当前功能梳理和问题清单。

### 第 2 阶段：功能优化和设计落地

- 前端接入设计素材。
- 修复主要中文乱码和明显 UI 问题。
- 优化文本情绪规则和情绪词典。
- 完成音频模型基本测试报告。
- 验证日记保存和导出流程。

### 第 3 阶段：展示和部署

- 准备一条完整 demo 流程。
- 确认本地和 Render 至少一个环境能稳定运行。
- 完成项目展示材料。
- 整理每位同学的贡献说明。

## 十、协作注意事项

- 设计稿不是最终成果，能被网页使用的素材、规则和组件才是可合并贡献。
- 技术同学不要等设计完全结束再开发，应先实现可替换的 UI 区块。
- 设计同学导出素材时优先使用 SVG 和 WebP，避免过大的 PNG。
- 所有人修改前先确认自己负责的文件范围，减少冲突。
- 不要提交 `.env`、私钥、证书、日志、本地数据库和缓存文件。
- 每次合并前至少做一次本地运行或语法检查。
